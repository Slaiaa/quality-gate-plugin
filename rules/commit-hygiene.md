Before committing code or suggesting a commit:

- Verify the code compiles/typechecks without errors
- Verify formatting matches the project's configured formatter
- Do not create "fix formatting" or "fix lint" follow-up commits -- fix issues in the original commit
- Do not commit generated files (dist/, build/, .next/) unless the project explicitly tracks them
- Do not commit files with conflict markers (<<<<<<, ======, >>>>>>)
